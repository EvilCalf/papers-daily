## 2. 研究目标（详细）
- **核心痛点**: 
    1. 现有策略语言（如 BDDs, SMT, NetKAT）假设规则条件为清晰布尔谓词（Crisp Boolean Predicate），无法处理基于嵌入相似度、领域分类器等概率 ML 信号。
    2. 当两个意图不相交的类别信号在同一查询上均超过阈值时，系统会静默地将请求路由至错误模型，编译器无任何警告。
    3. 缺乏对概率谓词冲突的可判定性理论分析，导致生产环境中存在隐蔽的路由错误风险。
- **研究问题**: 如何在基于概率 ML 信号的策略语言中，实现冲突的检测与预防，同时保证无需重新训练模型？
- **研究目标**: 
    - 建立概率谓词冲突的三级可判定性层次结构（Three-level Decidability Hierarchy）。
    - 证明嵌入冲突可归约为球冠交集（Spherical Cap Intersection）问题。
    - 设计基于温度缩放 Softmax 的机制，将嵌入空间划分为无冲突的 Voronoi 区域。
    - 在 Semantic Router DSL 中实现该机制并验证其在 LLM 推理路由中的有效性。
- **应用场景**: 大语言模型（LLM）推理路由、语义基于角色的访问控制（Semantic RBAC）、API 网关策略管理。

## 3. 方法概述（技术细节）
- **核心思想**: 将传统的独立阈值判断（Independent Thresholding）替换为温度缩放 Softmax（Temperature-scaled Softmax），利用几何特性强制互斥。
- **架构设计**: 
    - 系统基于 Semantic Router DSL 构建，包含信号定义（SIGNAL）、路由规则（ROUTE）及后端绑定（BACKEND）。
    - 核心决策模块不依赖外部分类器重训练，而是在推理时动态调整概率分布。
    - 嵌入空间被映射为单位超球面，决策边界由 Softmax 的温度参数 $\tau$ 控制。
- **关键创新**: 
    1. **三级可判定性层次**: 
       - 清晰冲突（Crisp Conflicts）：通过 SAT 求解器可判定（Decidable）。
       - 嵌入冲突（Embedding Conflicts）：归约为球冠交集问题，几何上可解。
       - 分类器冲突（Classifier Conflicts）：若无分布知识则不可判定（Undecidable）。
    2. **Voronoi 区域划分**: 
       - 传统方法：$P(class_i) > \theta$ 导致重叠区域。
       - 本方法：$\text{softmax}(z_i / \tau)$ 确保 $\sum P_i = 1$，且 $\arg\max$ 唯一。
       - 数学表达：决策边界由 $z_i - z_j = 0$ 定义，形成凸多面体区域。
    3. **零重训练机制**: 
       - 无需反向传播更新模型权重，仅调整推理时的温度参数 $\tau$。
       - 计算开销增加可忽略不计（$O(N)$ 复杂度，N 为类别数）。
- **与现有方法的区别**: 现有方法（如 NetKAT）依赖布尔逻辑，无法处理连续向量空间；本方法直接在嵌入几何空间操作，解决了概率信号的重叠问题。

## 6. 优势（量化对比）
- **性能优势**: 相比独立阈值法，冲突发生率从“未知/非零”降低至 0%，消除了静默路由错误。
- **效率优势**: 推理延迟增加极小，仅增加一个 Softmax 计算层，参数量增加 0，训练时间减少 100%（无需重训）。
- **通用性优势**: 适用于任何基于嵌入相似度的路由系统，包括语义 RBAC 和 API 网关，不依赖特定模型架构。
- **理论优势**: 建立了完整的可判定性层次结构，明确了哪些冲突可解（嵌入）、哪些不可解（无分布知识的分类器），为系统设计提供了理论边界。

## 7. 局限性和风险（诚实评估）
- **技术局限**: 
    1. 分类器冲突（Classifier Conflicts）在缺乏分布知识的情况下被证明为不可判定，本方法无法解决此类所有冲突。
    2. 依赖嵌入空间的质量，若嵌入模型本身区分度低，Voronoi 区域虽无冲突但语义分类可能不准。
- **实验局限**: 提供的片段未包含大规模基准测试（如 GLUE, SuperGLUE）上的量化准确率对比，主要侧重于系统正确性而非任务性能。
- **应用风险**: 温度参数 $\tau$ 的选择需要调优，过低可能导致决策边界过于敏感，过高则可能模糊语义界限。
- **未来改进方向**: 探索如何结合分布知识解决分类器冲突问题，以及自动化温度参数 $\tau$ 的搜索策略。

- **对研究者的价值**: 提供了处理概率谓词冲突的新视角，将策略验证问题转化为几何问题（球冠交集），可启发后续形式化验证工作。
- **对工程师的价值**: 在构建 LLM 路由系统时，应避免使用独立阈值判断多个语义类别，改用互斥的 Softmax 机制以确保系统稳定性。
- **对行业的价值**: 为 API 网关和语义访问控制提供了安全基线，防止因策略冲突导致的权限绕过或数据泄露。

## 10. 简要结论（精华总结）
本文针对基于概率 ML 谓词的策略语言冲突问题，提出了一个三层可判定性框架，并通过温度缩放 Softmax 将嵌入空间划分为互斥的 Voronoi 区域，实现了理论上的 0% 冲突率。该方法无需模型重训练，已成功集成于 Semantic Router DSL，为 LLM 推理路由及语义安全策略提供了可靠的无冲突保障，显著提升了系统的可验证性与安全性。
