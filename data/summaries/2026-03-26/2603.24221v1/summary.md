## 2. 研究目标（详细）

- **核心痛点**:
    1.  **基础设施复杂性**: 数字基础设施的互连性和复杂性日益增加，使得可扩展且可靠的安全评估方法变得至关重要，传统手动测试难以覆盖。
    2.  **机器人系统特殊性**: 现代机器人是高度网络化的信息物理系统（Cyber-Physical Systems, CPS），部署于工业自动化等领域，其操作技术（OT）环境对安全性有特殊要求。
    3.  **上下文管理缺失**: 现有自动化测试缺乏有效的上下文管理和可追溯性，难以满足如《欧盟人工智能法案》（EU AI Act）等框架对人类监督的要求。
- **研究问题**: 如何利用大语言模型（LLM）构建一个既能自主执行渗透测试，又能保持状态可追溯、上下文有效的多智能体工作流，特别是在机器人操作系统（ROS/ROS2）环境中？
- **研究目标**:
    - 设计一个基于 LangGraph 的多智能体闭环架构，包含规划、执行和记忆三个核心代理。
    - 构建动态图基记忆系统，实时捕捉网络拓扑、通信通道及漏洞状态。
    - 在机器人 CTF 场景中实现 100% 的任务完成率，同时确保测试过程的可解释性。
    - 支持用户切换操作模式（如扫描或利用），以指导评估过程。
- **应用场景**: 工业自动化安全评估、物流机器人网络审计、自主服务机器人的漏洞扫描。

## 3. 方法概述（技术细节）

- **核心思想**: 该方法本质上是构建一个状态机驱动的多智能体协作系统，通过将环境状态映射为图结构记忆，解决 LLM 在长程任务中的上下文丢失问题，实现环境感知的自动化渗透。
- **架构设计**:
    系统基于 LangGraph 框架构建，采用闭环架构。核心包含三个协同代理：
    1.  **Planner（规划器）**: 负责生成结构化任务列表。
    2.  **Executor（执行器）**: 负责将任务翻译为可执行命令（如 Nmap, ROS  exploit）。
    3.  **Memory Agent（记忆代理）**: 负责审查完成任务并将提取信息存入图记忆（GraphMemory）。
    架构不依赖传统的 Transformer 层数调整，而是依赖代理间的状态转移逻辑。
- **关键创新**:
    1.  **环境感知的图记忆机制 (Environment-Grounded Graph Memory)**:
        不同于传统的向量数据库检索，该方法动态构建基于图的记忆。节点代表主机或服务，边代表连接或漏洞关系。
        逻辑表达为：$G_t = (V_t, E_t)$，其中 $V$ 包含 observable system state，$E$ 包含 communication channels。每次执行后，$G_{t+1} = \text{Update}(G_t, \text{Observation})$。
    2.  **结构化任务验证流程**:
        在规划阶段，生成的任务列表需经过预定义语法验证。例如，初始扫描任务被格式化为 JSON 对象，包含 `task_id`, `target`, `task_type` 等字段，确保执行器能无误解析。
        示例结构：`{"task_type": "nmap_scan", "target": "172.19.0.0/24"}`。
    3.  **多模式操作控制 (Operational Modes)**:
        系统支持用户在参与过程中切换模式（如 Scanning Mode 到 Exploitation Mode）。这引入了人类在回路（Human-in-the-loop）的控制变量，允许根据实时风险评估调整代理行为策略。
- **与现有方法的区别**: 现有自动化渗透测试工具（如自动扫描器）通常是无状态的或基于线性日志，缺乏对复杂拓扑的结构化理解；本方法通过图记忆显式建模系统状态，增强了长程任务中的上下文一致性。

## 6. 优势（量化对比）

- **性能优势**: 在机器人 CTF 场景下，任务成功率达到 100%，相比文献基准有显著提升（具体百分比未量化，但描述为"significantly exceeds"）。
- **效率优势**: 通过多智能体分工（规划、执行、记忆），避免了单一模型在处理复杂任务时的认知过载，提高了任务执行的结构化效率。
- **通用性优势**: 架构设计 tailored to Robotics-based systems，但基于 LangGraph 的通用性使其理论上可扩展至其他 IoT 或 CPS 领域。
- **理论优势**: 提出了“环境接地”（Environment-Grounded）的概念，将 LLM 的推理能力与具体的系统状态图绑定，解决了大模型在安全测试中常见的“幻觉”导致无效攻击的问题。

## 7. 局限性和风险（诚实评估）

- **技术局限**:
    1.  **样本量较小**: 实验仅进行了 n=5 次测试运行，统计显著性有待更大规模验证。
    2.  **依赖特定框架**: 目前主要针对 ROS/ROS2 环境，对非机器人系统的通用网络渗透测试能力未验证。
    3.  **LLM 依赖性**: 规划器和执行器的效果底层依赖于基座 LLM 的能力，若基座模型更新或变更，可能影响工作流稳定性。
- **实验局限**: 实验在专门的 CTF 场景中进行，可能与真实世界工业环境的复杂噪声和防御机制存在差距（Sim-to-Real Gap）。
- **应用风险**: 自动化渗透测试工具若被恶意使用，可能对机器人系统造成实际损害；需严格控制访问权限。
- **未来改进方向**: 增加测试样本量至 n>50；扩展支持更多类型的操作技术（OT）协议；引入强化学习优化规划策略。

本文提出了一种环境感知的多智能体工作流，通过动态图记忆机制解决了机器人系统自动化渗透测试中的上下文管理难题。实验在 ROS/ROS2 场景下实现了 100% 的任务完成率，显著超越了现有基准，同时保持了符合法规要求的可追溯性。该方法为复杂信息物理系统的安全评估提供了高可靠性、结构化的自动化解决方案，具有重要的工业应用前景。
