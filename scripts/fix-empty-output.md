# 每日论文推送系统故障修复报告

## 问题现象
2026-03-30 推送为空，检索到0篇论文。

## 根因分析
1. **主要原因：arXiv API 速率限制**
   - 多次请求返回 "Rate exceeded" 错误
   - 原有重试机制和等待时间不足
   - 缺少浏览器UA头，更容易被反爬虫识别

2. **次要原因：请求频率过高**
   - 原有类别间等待仅0.5秒，过于频繁
   - 超时时间60秒过短，API响应较慢时容易超时

## 已实施修复
### 1. 优化 `simple-arxiv-search.py`
- **增强速率限制检测**：同时检测429状态码和返回内容中的"Rate exceeded"
- **增加重试等待时间**：从60秒基础等待提升到120秒，指数退避系数从30秒提升到60秒
- **增加UA头**：模拟Chrome浏览器请求，降低被反爬虫拦截概率
- **延长超时时间**：从60秒增加到120秒
- **降低请求频率**：类别间等待从0.5秒增加到5秒

### 2. 长期优化建议
- 考虑使用arxiv镜像或第三方API源，避免单一API故障
- 增加请求失败后的降级逻辑，尝试使用更早的日期范围
- 配置监控告警，推送为空时自动通知管理员
